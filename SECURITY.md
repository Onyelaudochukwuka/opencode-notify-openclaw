# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 0.2.x   | :white_check_mark: |
| < 0.2   | :x:                |

## Reporting a Vulnerability

If you discover a security vulnerability in **opencode-notify-openclaw**, please report it responsibly.

**Do not open a public GitHub issue for security vulnerabilities.**

Instead, please email **[udochukwukaonyela@gmail.com](mailto:udochukwukaonyela@gmail.com)** with the following details:

- A description of the vulnerability
- Steps to reproduce the issue
- The potential impact
- Any suggested fixes (if applicable)

## What to Expect

- **Acknowledgement** within **48 hours** of your report.
- A follow-up with an assessment and timeline within **5 business days**.
- A fix released as soon as practically possible, depending on severity.
- Credit in the release notes (unless you prefer to remain anonymous).

## Scope

This policy covers the `opencode-notify-openclaw` npm package and its source code. Issues in upstream dependencies (`@opencode-ai/plugin`, `@opencode-ai/sdk`) should be reported to their respective maintainers.

## Best Practices for Users

- Always use the latest supported version.
- Restrict network access for the reply server to trusted interfaces only.

Thank you for helping keep this project and its users safe.
